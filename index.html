THE FRAMEWORK

The Continuous Loop — 3 Moves

🎨
① PM Vibe Codes Prototype
Non-engineer PMs use AI to prototype concepts — working UIs, not wireframes. Align stakeholders with something they can touch, not read.
🤖
② Engineer + Agents
Engineers orchestrate AI agents on production codebase — scaffolding, boilerplate, tests, migrations. Human judgment on architecture. 10–12 weeks → 2 weeks.
🔍
③ PM Explores Next
PM uses agents to explore the production codebase — discover data patterns, prioritize backlog, vibe-code the next concept. Loop restarts.
💡 Think of it as a relay race — each step hands the baton to the next. AI is the track that makes every handoff faster.
STEP 1 — PM VIBE CODING

From Spec Document to Working Prototype in Hours

🧪 Your Vibe-Coding Recipe
① Start with the user story — not a feature list
② Describe the outcome: "I want a dashboard builder using e-charts for display, connected to MCP, with drag-and-drop charts and configurable features abc"
③ Iterate with natural language to add more features
④ Your prototype IS the spec — show it, don't document it
🎨Artifacts You Can Vibe-Code
• Interactive prototypes — clickable UIs, not wireframes
• Data dashboards — with mock data stakeholders can explore
• Workflow demos — multi-step flows users can walk through
• Concept comparisons — A/B variants to discuss trade-offs
• Journey maps — visual paths through your product

YOUR PLAYBOOK — WHAT TO OUTSOURCE

Intelligent Laziness & The DRAG Framework

Not everything deserves your full attention. Some tasks have a capped payoff — no matter how much effort you invest, returns plateau. Others have uncapped payoff — the more you think, the better. Know the difference.

→ Let AI handle: status reports, boilerplate, data cleanup, meeting summaries, translations
→ YOU own: product strategy, system architecture, UX decisions, user research, data modeling
📝
D
Drafting
🔍
R
Research
📊
A
Analysis
⚙️
G
Grunt Work
For PMs & Ops (DRAG examples)
D: Draft PRDs, release notes, stakeholder updates • R: Compare competitor features, summarize user feedback • A: Analyze survey responses, usage patterns • G: Translate docs, format reports, fill templates
For Engineers & Designers (DRAG examples)
D: Scaffold components, API stubs, test files • R: Compare libraries, read changelogs, benchmark tools • A: Error log patterns, PR review diffs, perf profiles • G: Unit tests, SQL conversions, mock data, style migrations
🍺 AI = drunk genius. Brilliant but unreliable. Make sure YOU drive the car. DRAG = the stuff you can safely hand to the passenger. Everything else — you steer.
THE PRINCIPLE

Using AI Intelligently — Not Blindly

Speed without judgment is just faster mistakes. Here's how each step of our loop maps to intelligent AI use.

🎨
PM Vibe Code
Zone 1 (DRAG): Drafting UI, generating boilerplate. PM's product sense (Zone 2) decides WHAT to build — AI handles HOW to scaffold it.
🤖
Engineer + Agents
Intelligent Hill: Engineers climb from zero-shot to agent orchestration. Key: provide context (few-shot), decompose tasks (Zone 1.5), keep architecture decisions human.
🔍
PM Explores
Intelligent Fool: PM asks "naive" questions to the codebase via agents — building the same mental model that end-users have. Foolish questions → better product.
🧠 AI amplifies what's already there. If you have good judgment, AI makes you dangerous. If you don't, AI makes you dangerously wrong — faster.
